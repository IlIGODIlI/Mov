# API Endpoint Documentation

This document describes all available REST endpoints of the Bus Transport Management System, including the required roles, authentication headers, request body schemas, and response formats.

---

## Authentication & Headers

- All role-protected API requests must include an **Authorization Header**:
  ```http
  Authorization: Bearer <your_jwt_token_here>
  ```
- Public endpoints (`/login`, `/register`) do not require authentication headers.

---

## Auth Endpoints (Public)

### 1. User Registration
Creates a new account in the system database.

- **URL**: `/register`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "email": "student@example.com",
    "password": "securepassword123",
    "role": "STUDENT",
    "name": "Jane Doe"
  }
  ```
- **Response** (JSON - Saved User Entity):
  ```json
  {
    "id": 1,
    "name": "Jane Doe",
    "email": "student@example.com",
    "password": "securepassword123",
    "role": "STUDENT"
  }
  ```

### 2. User Login
Authenticates user credentials and generates a signed JWT.

- **URL**: `/login`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "email": "student@example.com",
    "password": "securepassword123",
    "role": "STUDENT"
  }
  ```
- **Response Success** (Text - JWT Token):
  ```text
  eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJzdHVkZW50QGV4YW1wbGUuY29tIiwicm9sZSI6IlNUVURFTlQiLCJpYXQiOjE2ODg5OTAwMDAsImV4cCI6MTY4OTA3NjQwMH0...
  ```
- **Response Errors** (Text):
  - `"User not found"` (404)
  - `"Incorrect role"` (403)
  - `"Invalid password"` (401)

## Admin Endpoints (Requires `ADMIN` role)

### 1. Create a Route
Creates a new route metadata record. The route code is auto-generated by the server.

- **URL**: `/admin/createRoute`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "routeName": "Route A - North Campus"
  }
  ```
- **Response** (JSON - Route Details):
  ```json
  {
    "id": 5,
    "routeName": "Route A - North Campus",
    "routeCode": "BUS-H2K9",
    "createdBy": null,
    "startLat": 0.0,
    "startLng": 0.0,
    "endLat": 0.0,
    "endLng": 0.0
  }
  ```

### 2. Assign Route
Assigns a route code to a specific student or driver.

- **URL**: `/admin/assignRoute`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "email": "driver@example.com",
    "routeCode": "BUS-H2K9"
  }
  ```
- **Response Success** (Text):
  ```text
  "Route BUS-H2K9 assigned to driver driver@example.com"
  ```
- **Response Errors** (Text):
  - `"User not found"`
  - `"Invalid role for assignment"`

### 3. Update Route Coordinates
Updates the start and end coordinates of a route path.

- **URL**: `/admin/updateRoutePosition`
- **Method**: `PUT`
- **Request Body**:
  ```json
  {
    "routeCode": "BUS-H2K9",
    "startLat": "28.6139",
    "startLng": "77.2090",
    "endLat": "28.6304",
    "endLng": "77.2177"
  }
  ```
- **Response Success** (Text):
  ```text
  "Route coordinates updated successfully"
  ```
- **Response Errors** (Text):
  - `"Route not found"`
  - `"Invalid coordinate format: ..."`

### 4. Fetch All Users
Lists all users registered in the database.

- **URL**: `/admin/allUsers`
- **Method**: `GET`
- **Response** (JSON Array):
  ```json
  [
    {
      "id": 1,
      "name": "Jane Doe",
      "email": "student@example.com",
      "password": "securepassword123",
      "role": "STUDENT"
    }
  ]
  ```

### 5. Fetch All Routes
Lists all route definitions.

- **URL**: `/admin/allRoutes`
- **Method**: `GET`
- **Response** (JSON Array):
  ```json
  [
    {
      "id": 5,
      "routeName": "Route A - North Campus",
      "routeCode": "BUS-H2K9",
      "createdBy": null,
      "startLat": 28.6139,
      "startLng": 77.2090,
      "endLat": 28.6304,
      "endLng": 77.2177
    }
  ]
  ```

### 6. Fetch All Bus Locations
Retrieves active tracking positions of all vehicles.

- **URL**: `/admin/allLocations`
- **Method**: `GET`
- **Response** (JSON Array):
  ```json
  [
    {
      "id": 1,
      "routeCode": "BUS-H2K9",
      "latitude": 28.6152,
      "longitude": 77.2110
    }
  ]
  ```

## Driver Endpoints (Requires `DRIVER` role)

### 1. Start Trip
Logs a driver starting a route. Marks the trip record as active.

- **URL**: `/driver/startTrip`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "driverEmail": "driver@example.com",
    "routeCode": "BUS-H2K9"
  }
  ```
- **Response** (Text):
  ```text
  "Trip started successfully"
  ```

### 2. Update Location
Updates the current coordinates of a bus on an active route.

- **URL**: `/driver/updateLocation`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "routeCode": "BUS-H2K9",
    "latitude": 28.6152,
    "longitude": 77.2110
  }
  ```
- **Response** (Text):
  ```text
  "Location updated"
  ```

## Student Endpoints (Requires `STUDENT` role)

### 1. Join Route
Registers the logged-in student to a specific route. Uses the student's email extracted from the session JWT.

- **URL**: `/student/joinRoute`
- **Method**: `POST`
- **Request Body**:
  ```json
  {
    "routeCode": "BUS-H2K9"
  }
  ```
- **Response** (Text):
  ```text
  "Successfully joined route"
  ```

### 2. Fetch Bus Location
Retrieves the coordinates of the bus on a specific route code.

- **URL**: `/student/busLocation/{routeCode}`
- **Method**: `GET`
- **Response** (JSON):
  ```json
  {
    "id": 1,
    "routeCode": "BUS-H2K9",
    "latitude": 28.6152,
    "longitude": 77.2110
  }
  ```

### 3. Fetch Assigned Route
Retrieves the route details that the current student has joined.

- **URL**: `/student/myRoute`
- **Method**: `GET`
- **Response** (JSON):
  ```json
  {
    "id": 12,
    "studentEmail": "student@example.com",
    "routeCode": "BUS-H2K9"
  }
  ```

### 4. Fetch Bus ETA
Calculates the estimated minutes until arrival based on Euclidean distance to the route's destination at a constant velocity proxy.

- **URL**: `/student/eta/{routeCode}`
- **Method**: `GET`
- **Response** (Numeric):
  ```text
  12.45
  ```

### 5. Fetch Route Details
Retrieves aggregate metrics and metadata for a specific route.

- **URL**: `/student/routeDetails/{routeCode}`
- **Method**: `GET`
- **Response** (JSON):
  ```json
  {
    "studentCount": 15,
    "driverEmail": "driver@example.com",
    "startLat": 28.6139,
    "startLng": 77.2090,
    "endLat": 28.6304,
    "endLng": 77.2177
  }
  ```
